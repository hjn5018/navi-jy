# NAVI UI/UX 와이어프레임 설계

NAVI는 기본적으로 '음성'이 주 인터페이스(VUI, Voice User Interface)입니다. 화면은 보조적인 피드백을 주며, 시각 장애인 외의 일반 사용자나 저시력자를 위해 직관적이고 크고 뚜렷한 UI를 제공해야 합니다.

## 1. 화면 모드 구분
NAVI 프로그램은 확장성을 고려하여 두 가지 모드의 UI를 가집니다.
1. **오버레이 위젯 (Overlay Widget)**: 평상시 화면 귀퉁이나 중앙 상단에 매우 작게 플로팅되어 떠있는 형태. (화면을 가리지 않는 컴팩트한 비서 형태)
2. **메인 윈도우 모드 (Main Window)**: 대화 기록, 작업 로그, 설정 등을 확인할 수 있는 확장된 대시보드.

## 2. 와이어프레임 플로우

### A. 플로팅 오버레이 상태 (대기 및 동작 중)
- **UI 요소**: 마이크 아이콘 모양의 원형 버튼 및 상태 표시 링 (투명도 적용 가능)
- **인터랙션**: 상태에 따라 색상과 애니메이션 변화
  - **대기 중 (Idle)**: 창백한 회색 또는 은은한 파란색의 부유하는(숨쉬는) 애니메이션.
  - **호출됨/듣는 중 (Listening)**: 음성 파고(Waveform)를 나타내는 오디오 스펙트럼 애니메이션. 밝은 초록색/파란색 파티클 이펙트.
  - **처리 중 (Processing)**: 부드럽게 돌아가는 로딩 스피너 애니메이션.
  - **말하는 중 (Speaking/TTS)**: 중심에서 바깥으로 원형 파동(Ripple)이 뻗어나가는 이펙트.

### B. 메인 윈도우 (채팅 & 로그 대시보드)
- **Header**: 애플리케이션 로고(`NAVI`), 화면 핀(고정) 토글 버튼, 설정(톱니바퀴) 버튼, 최소화/닫기 
- **Body (Chat UI)**: 카카오톡이나 챗봇과 유사한 대화 레이아웃
  - **사용자 영역 (우측)**: "유튜브에서 고양이가 춤추는 영상 찾아줘"
  - **NAVI 영역 (좌측)**: "유튜브에서 영상을 찾아봤어요. 첫 번째 영상은 '스텝 밟는 고양이' (3:15 소요) 입니다. 재생할까요?"
  - **시스템 로그 영역 (중앙/선택적 표시)**: 
    - `💻 [System] 브라우저 실행 중 (Playwright)`
    - `🔍 [WebAgent] 유튜브 검색 쿼리 전송 및 결과 파싱 완료`
- **Footer**: 명령어를 텍스트로도 입력할 수 있는 텍스트 필드와 전송 버튼, 마이크 토글 버튼

## 3. 크로스 플랫폼 (스마트폰 및 Mac) UI 대응 전략
- 스마트폰(iOS/Android)에서는 Siri나 Google Assistant처럼 앱 하단에서 올라오는 Bottom Sheet 스타일 채택.
- 중앙에 커다란 상태 애니메이션 위젯을 배치하고 위로 스와이프 시 대화 히스토리에 접근하는 원-페이지 레이아웃 구조를 권장합니다.
