# ✅ NAVI QA 및 테스트 가이드 (QA & Test Plan)

시스템의 안정성과 정확성을 검증하기 위한 테스트 항목들을 정리했습니다.

## 1. 핵심 기능 테스트 (Core Functionality)
| 테스트 항목 | 기대 결과 | 상태 |
| :--- | :--- | :--- |
| **의도 분석 (Intent Routing)** | 다양한 자연어 입력(예: "유튜브에서 ~ 찾아줘", "메모장 열어줘")에 대해 정확한 에이전트를 매칭하는가? | [ ] |
| **유튜브 검색 (Web Agent)** | 검색 결과 3개가 제목, 링크, 채널 정보와 함께 정상적으로 추출되는가? | [ ] |
| **메모장 제어 (OS Agent)** | 실제 Windows 메모장 창이 활성화되고 텍스트가 정확히 입력되는가? | [ ] |
| **WebSocket 연결** | 프론트엔드-백엔드 간 실시간 상태(LIVE/OFFLINE) 표시가 동기화되는가? | [ ] |

## 2. 성능 및 안정성 테스트 (Performance & Stability)
- **Latentcy Check**: 음성/텍스트 전송 후 결과가 3초 이내에 출력되는가?
- **Resource Usage**: 백엔드 브라우저(Playwright)가 작업 완료 후 메모리를 정상적으로 반환하는가?
- **Auto-Reconnect**: 백엔드 서버가 강제 종료 후 재실행되었을 때 클라이언트가 자동으로 재연결되는가?

## 3. 보안 테스트 (Security)
- **Data Masking**: "내 이메일은 test@google.com이야"라고 입력했을 때 백엔드 로그에 `[EMAIL_MASKED]`로 기록되는가?
- **API Key Error**: `.env`에 키가 없을 때 사용자에게 명확한 에러 메시지를 표시하는가?

## 4. UI/UX 테스트
- **Responsive Layout**: 대시보드 모드와 컴팩트 모드에서 UI가 깨지지 않고 정상적으로 렌더링되는가?
- **Scroll Behavior**: 메시지가 많아질 때 자동으로 하단 스크롤이 유지되는가?
- **Animation**: 모드 전환 시 부드러운 트랜지션이 발생하는가?

## 5. 단계별 테스트 시나리오 (Walkthrough)
1. 서버 및 클라이언트 실행
2. "안녕 나비?" 입력 (기본 인사 확인)
3. "유튜브에서 신나는 음악 찾아줘" (웹 결과 확인)
4. "메모장에 오늘 일기 써줘" (OS 제어 확인)
5. 오른쪽 상단 '컴팩트 모드' 버튼 클릭 (UI 전환 확인)
