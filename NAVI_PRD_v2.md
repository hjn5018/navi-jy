# 개선된 NAVI 제품 요구사항 정의서 (PRD v2.0)

## 1. 프로젝트 개요
**프로젝트 명:** NAVI (Navigation & Voice Interface)
**목적:** 시각 장애인, 저시력자 및 일반 사용자의 접근성 향상과 편의를 위한 음성 명령 기반 브라우저 및 OS 제어 AI 에이전트.
**확장 목표:** Windows 데스크톱 환경을 시작으로, 추후 Mac OS, 스마트폰 애플리케이션(iOS/Android) 및 웹 환경으로 확장 가능한 크로스 플랫폼 아키텍처 지향.

## 2. 주요 기능 및 특징
1. **상시 음성 호출 (Wake Word)**
   - 백그라운드 환경에서 동작하며, 단축키나 특정 웨이크 워드("나비야")를 통해 메인 에이전트 호출.
2. **자연어 기반 명령 처리**
   - 사용자 음성을 텍스트로 변환(STT).
   - LLM(GPT-4o-mini 등)을 사용하여 의도 파악 후 작업 계획(Plan) 생성 및 적절한 하위 Agent에 작업 할당.
3. **크로스 플랫폼 및 OS 제어**
   - 웹 브라우저 제어 (Playwright 활용).
   - PC 제어 (Windows UI Automation 활용, 추후 Mac OS accessibility API 대응).
4. **결과 피드백 및 후속 동작**
   - 실행 결과를 바탕으로 작업 완수 여부 검증 및 예외 처리.
   - 텍스트 결과를 음성으로 변환하여 자연스럽게 안내(TTS).
   - 작업 완료 후 후속 행동 옵션 제시("첫 번째 영상을 재생할까요?").
5. **보안 및 프라이버시 처리**
   - 화면 내 개인정보(비밀번호 등) 식별 시 음성 읽기 생략 및 로그 마스킹 처리.

## 3. 핵심 사용 시나리오 (예시)
- **콘텐츠 검색 및 재생 (유튜브)**
  - 사용자: "유튜브에서 고양이가 춤추는 영상 찾아줘"
  - 시스템: 브라우저 실행 -> 검색 -> 결과 파싱 -> 요약 안내 및 후속 조치("첫 번째 영상은 ... 입니다. 재생할까요?")
- **경로 탐색 (네이버 지도)**
  - 사용자: "서울역에서 폴리텍대학 인천캠퍼스로 가는 지하철 경로 알려줘"
  - 시스템: 지도 접속 -> 경로 탐색 -> 대중교통 정보 추출 -> 요약 안내 (소요시간, 환승 횟수)
- **앱 제어 (메모장)**
  - 사용자: "메모장 열고 오늘 할 일 적어서 저장해줘"
  - 시스템: 메모장 앱 실행 -> 텍스트 입력 -> 파일 저장 대화상자 제어 -> 완료 결과 안내

## 4. 향후 확장성
- **모바일/플랫폼 확장:** API 서버 분리를 통해 모바일 앱에서 음성을 전송받고, 서버에서 처리된 결과를 반환하는 클라이언트-서버 구조 도입.
