# Navi — 시각장애인을 위한 음성 기반 PC 자동화 어시스턴트

## 프로젝트 개요

**Navi**는 시각장애인 및 저시력 사용자가 **음성만으로 PC를 조작**할 수 있도록 돕는 Windows 데스크톱 어시스턴트다.

사용자가 말하면 AI가 듣고(STT), 의도를 분석하고(LLM), 브라우저와 응용프로그램을 직접 조작하고, 결과를 음성으로 알려주는(TTS) **End-to-End 음성 자동화 시스템**이다.

---

## 대상 사용자

| 구분 | 설명 |
|------|------|
| **1차 타겟** | 전맹(전체 시각장애인) — 화면을 전혀 볼 수 없는 사용자 |
| **2차 타겟** | 저시력 사용자 — 화면 확대, 고대비 등 보조가 필요한 사용자 |
| **3차 타겟** | 상지 장애 사용자 — 키보드/마우스 사용이 어려운 사용자 |
| **잠재 타겟** | 멀티태스킹 일반 사용자 — 음성으로 빠른 조작을 원하는 사용자 |

## 제품 형태

```
┌─────────────────────────────────────────────┐
│              Windows System Tray            │
│  ┌─────┐                                    │
│  │ 🔊  │  ← 백그라운드 음성 감지 (항상 실행)   │
│  └──┬──┘                                    │
│     │  wake word 또는 핫키 감지               │
│     ▼                                       │
│  ┌─────────────────────────────┐            │
│  │     Navi 메인 팝업 창        │            │
│  │  ┌───────────────────────┐  │            │
│  │  │  🎙️ 듣고 있습니다...   │  │            │
│  │  │                       │  │            │
│  │  │  "유튜브에서 고양이     │  │            │
│  │  │   영상 찾아줘"         │  │            │
│  │  │                       │  │            │
│  │  │  ⏳ 실행 중...         │  │            │
│  │  │  ✅ 검색 완료!         │  │            │
│  │  └───────────────────────┘  │            │
│  └─────────────────────────────┘            │
└─────────────────────────────────────────────┘
```

- **시스템 트레이**: 백그라운드에서 항상 음성 감지
- **팝업 창**: 명령 감지 시 활성화되어 상호작용
- **자동 최소화**: 작업 완료 후 트레이로 복귀

## 기술 스택

| 계층 | 기술 | 역할 |
|------|------|------|
| **Frontend** | Flutter (Windows Desktop) | UI, 시스템 트레이, 사용자 상호작용 |
| **Backend API** | FastAPI (Python) | LLM 연동, Intent 분석, 작업 계획 수립 |
| **Local Automation** | Python (Playwright + pyautogui + Win32 API) | 브라우저/앱 자동화 실행 |
| **STT** | Google STT / Whisper / Azure STT | 음성 → 텍스트 변환 |
| **LLM** | GPT-4 / Gemini / Claude (유료) 또는 Gemma / Llama (무료) | 자연어 이해 및 작업 의미 분석 |
| **TTS** | Google TTS / Azure TTS / Edge TTS | 텍스트 → 음성 변환 |

## 기존 작업 현황

현재 프로젝트에는 이미 상당한 양의 코드와 문서가 있다.

### `Navi_VoiceNavigator_Front/` — Flutter 프론트엔드
- Day 0 ~ Day 24까지 **24일간의 개발 튜토리얼** 기록
- Pattern Task 기반 자동화 엔진 연동 완료
- 시스템 트레이 팝업, 접근성 UI 구현
- Demo 모드 / Connected 모드 분리 구조
- Riverpod 상태 관리, Pretendard 한글 폰트 적용

### `PlannerLLM/` — FastAPI 백엔드
- Intent Parser (LLM 기반 의도 분석)
- OpenAI API 연동
- FastAPI 라우터 구조

### 핵심 아키텍처 성과
- **Legacy JSON Plan** → **Pattern Task** 전환 완료
- LLM이 저수준 step 대신 `task_type + slots`만 반환하는 구조
- 로컬 Pattern Executor가 화면을 보고 직접 요소를 바인딩하는 구조
- Recovery 메커니즘 (실패 시 로컬 복구)

---

## 프로젝트 비전

> **"음성만으로 PC의 모든 것을 할 수 있는 세상"**

Navi는 시각장애인이 비장애인과 동일한 수준으로 PC를 사용할 수 있도록 하는 것을 최종 목표로 한다.
브라우저 자동화에서 시작하여, Windows 응용프로그램 전체로 자동화 범위를 확장하고,
최종적으로는 사용자의 **디지털 비서**로서 기능하는 것을 지향한다.
